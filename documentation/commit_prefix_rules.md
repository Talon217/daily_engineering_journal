# **feat:** *(Feature)*
> **Meaning:** You added a new piece of functionality. <br>
> **Usage:** "Adding a bot command that responds to greetings." <br>

# **fix:** *(Bug Fix)*
> **Meaning:** You corrected a bug or a broken piece of code. <br>
> **Usage:** "Repairing the bot's inability to connect to the Discord API." <br>

# **docs:** *(Documentation)*
> **Meaning:** You changed documentation (READMEs, your log.md, or code comments). This does not touch your functional code. <br>
> **Usage:** "Adding the log template to log.md." <br>

# **chore:** *(Chores/Maintenance)*
> **Meaning:** Boring but necessary tasks that don't affect the product's features or fix bugs. <br>
> **Usage:** "Updating the .gitignore or upgrading a dependency version." <br>

# **refactor:** *(Code Cleanup)*
> **Meaning:**  Changing existing code without changing how it works (e.g., making it faster or cleaner). <br>
> **Usage:** "Renaming a poorly named variable or simplifying a function." <br>
